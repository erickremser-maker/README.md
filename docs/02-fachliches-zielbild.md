# 02 Fachliches Zielbild

## Leitprinzip

Das Projektportal ist nicht eine weitere Liste, sondern der verbindliche Arbeitsort für technische Projekte. Alle führenden Projektinformationen werden strukturiert im Portal gepflegt. Dokumente liegen in SharePoint, sind aber aus der Projektakte heraus erreichbar und fachlich verschlagwortet.

## Zielgruppen und Nutzen

| Zielgruppe | Hauptbedarf | Portalnutzen |
|---|---|---|
| Management | Portfolioüberblick, Risiken, Eskalationen, Budget, Termine | Einheitliche Ampeln und Dashboards. |
| Auftraggeber | Zielerreichung, Budgetrahmen, Entscheidungen | Entscheidungs- und Eskalationslog. |
| Projektmanager | Steuerung von Phasen, Rollen, Terminen, Risiken, Budget, Stunden | Eine Projektakte statt paralleler Listen. |
| Planungsingenieur | Planungsstand, technische Unterlagen, Nachbesserungen | Klare Aufgaben und Dokumentenpakete. |
| Planprüfer | Prüffähige Unterlagen, Mängel, Prüfergebnis | Strukturierter Planprüfprozess. |
| Bauvorlagenberechtigter / BVB | Freigaberelevante Unterlagen, Auflagen, Entscheidung | Nachvollziehbare Freigabehistorie. |
| Realisierungsverantwortlicher | Umsetzung, Dienstleister, Abnahmen, Inbetriebnahme | Klare Termin- und Mängelsteuerung. |
| Dokumentationsverantwortlicher | As-built, Abschlussunterlagen, Archivierung | Vollständigkeitscheck je Projekt. |
| Externer Dienstleister | Aufgaben, Dokumente, Rückfragen | Eingeschränkter Zugriff auf relevante Inhalte. |

## Führende Informationen

| Information | Führendes System | Begründung |
|---|---|---|
| Projektstammdaten | Dataverse | Strukturierte Auswertung und Governance. |
| Rollen und Verantwortlichkeiten | Dataverse | Rollen sind projektbezogen und historisierbar. |
| Phasenstatus | Dataverse | Verbindliche Projektsteuerung. |
| Planprüfstatus | Dataverse | Prüfung darf nicht nur im Protokoll stehen. |
| BVB-Freigabestatus | Dataverse | Freigabe/Auflagen müssen steuerbar sein. |
| Termine und Meilensteine | Dataverse | Baseline, aktuelle Termine und Historie. |
| Terminverschiebungen | Dataverse | Pflichtbegründung und Genehmigungsworkflow. |
| Budget und Stunden | Dataverse / Import aus Quellsystem | Portal zeigt Führungs- und Steuerungssicht. |
| Dokumentdateien | SharePoint | Versionierung und Dokumentmanagement. |
| Dokumentmetadaten | Dataverse oder SharePoint-Spalten | Suche, Filter und Vollständigkeitslogik. |
| Benachrichtigungen | Teams/Outlook | Nur Hinweis, keine führende Ablage. |

## Fachlicher Scope

| Bestandteil | Im MVP | Später |
|---|---|---|
| Projektakte | Ja | Erweiterung um Automatisierung/KI. |
| Rollenmatrix | Ja | Skill- und Kapazitätsplanung. |
| Phasenmodell | Ja | phasenspezifische Workflows. |
| Planprüfung | Ja | digitale Kommentierung tiefer integrieren. |
| BVB-Freigabe | Ja | qualifizierte Signatur prüfen. |
| Budget/Stunden | Ja | ERP-/Zeiterfassungsintegration. |
| Dokumente | Ja | DMS-/CAD-Integration. |
| Dashboards | Ja | Management-Summaries und Trendanalysen. |
| Externe Zusammenarbeit | eingeschränkt | dediziertes Extranet. |
